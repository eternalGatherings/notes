4. Example test cases

I.Basic robot file example

*** Settings ***
Library    SeleniumLibrary

*** Variables ***
${URL}     https://example.com
${BROWSER}     chrome

*** Test Cases ***
Open Browser And Check Title
    Open Browser    ${URL}    ${BROWSER}
    Title Should Be    Example Domain
    Close Browser

II.Basic example for Data Driven Test

*** Settings ***
Library    SeleniumLibrary

*** Variables ***
${URL}         https://example.com/login
${BROWSER}     chrome

*** Test Cases ***
Login With Valid Credentials
    [Template]    Perform Login Test
    user1    pass123
    admin    admin123
    testuser    test456

*** Keywords ***
Perform Login Test
    [Arguments]    ${username}    ${password}
    Open Browser    ${URL}    ${BROWSER}
    Input Text      id:username    ${username}
    Input Text      id:password    ${password}
    Click Button    id:login
    Page Should Contain    Welcome
    Close Browser

6.Assertions

What Are Assertions in Robot Framework?

  • An assertion is a keyword that fails the test if a condition is not met.

Most assertions are built-in or come from libraries like:

  • BuiltIn (comes by default)
  • SeleniumLibrary
  • Collections
  • String
  • etc.

I.Example assertion script

*** Variables ***
${TITLE}    Welcome Page

*** Test Cases ***
Title Should Be Correct
    ${actual_title}=    Get Title
    Should Be Equal     ${actual_title}    ${TITLE}

II.Failing a Test Manually

If you want to fail a test intentionally:

Fail    This is a custom failure message

III.Types of assertions

a. Should Be Equal

Should Be Equal    Hello    Hello
Should Be Equal    ${x}     5

Case-sensitive. Use Should Be Equal As Strings or Should Be Equal As Numbers for stricter types.


b. Should Contain

Should Contain    Welcome to Robot    Robot

c. Should Start With / Should End With

Should Start With    Hello, World!    Hello
Should End With      file.txt         .txt

d. Should Be True / Should Be False 

Should Be True    ${value} > 10
Should Be False   ${is_logged_in}

e. Page Should Contain / Element Should Be Visible (Selenium)

Page Should Contain            Welcome
Page Should Contain Element    id=submit
Element Should Be Visible      css=.alert
Element Should Not Be Visible  id=loading

f. Checkbox / Radio Assertions

Checkbox Should Be Selected    id=accept
Radio Button Should Be Set To  gender    female

g. List Assertions (from Collections library)

List Should Contain Value     ${items}    Apple
List Should Not Contain Value    ${items}    Banana
Length Should Be              ${items}    3

You need to import Collections library:

Library    Collections

h. String Assertions (from String library)

Should Be Lower Case    robot
Should Be Upper Case    ROBOT

i. Other assertions

Title Should Be   Home Page // To validate the page title
Page Should Contain    Welcome // This is a Robot Framework assertion that checks whether the current web page contains the text "Welcome" anywhere in its visible content.
Page Should Not Contain    Welcome // This passes only if the word "Welcome" is not present anywhere in the visible text on the page.
